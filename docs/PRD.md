# Star Project PRD

## Product Name

Star Project

## Product Type

Website + PWA web app MVP for project-based career discovery.

## Primary Market

Kassel, Germany.

## Primary Users

1. Public visitors, mostly parents/guardians.
2. Registered parent-managed family accounts.
3. Children aged 4–18 using the account under guardian supervision.
4. Super Admin for content, payment, user, and submission operations.

## Product Positioning

Star Project helps children discover future careers by completing real-world projects, working in teams, practicing project management, submitting evidence, and earning stars through reviewed outcomes.

Star Project is not a passive video-course platform. It is a project-based career discovery platform where children choose a career category, join or invite a team member, complete a 4–5 week challenge, submit a report and video proof, receive review, earn stars, unlock levels, and build a long-term achievement profile.

## Out of Scope for MVP

The MVP will not include:

1. Separate child login
2. Free-form chat between children
3. Native Android or iOS app
4. Public child media sharing without parent consent
5. Expert contributor dashboard
6. Direct expert access to child data
7. Fully automated video review
8. School/institution dashboard
9. Marketplace
10. Complex gamification beyond stars, levels, achievements, and certificates
11. Advanced AI mentor
12. Multi-child account under one guardian account

## Account Model

One account represents one guardian and one child.

Rules:

1. The guardian manages the account.
2. The child may use the account under guardian supervision.
3. One account contains one child profile only.
4. If a guardian has two children, they must create two separate accounts.
5. No separate child login is required in the MVP.
6. Each child receives a system-generated Student ID.
7. Student ID is used for team invitations.
8. Email and payment belong to the guardian.

1. Real projects, not passive lessons.

## Project Rules

1. A project is the primary learning unit.
2. Regular projects usually last 4 weeks.
3. Advanced projects may last 5 weeks.
4. Every project must have category, level, age recommendation, duration, tasks, safety labels, materials, and star reward.
5. Regular projects reward 1 star.
6. Mythical projects reward 2–3 stars.
7. Projects must follow a research-based learning cycle:
   - Week 1: Research and validation
   - Week 2: Build and create
   - Week 3: Test and validate
   - Week 4: Improve and finalize
   - Week 5: Optional showcase/reflection for advanced projects

## Team Rules

1. Minimum 2 registered users are required to start a project.
2. Solo start is not allowed.
3. Invitations are sent by Student ID only.
4. Only registered users can be invited.
5. Pending invitations expire after 7 days.
6. Project cannot start until minimum team size is reached.
7. No free-form chat is allowed in MVP.
8. Communication is limited to system messages, invitations, deadline reminders, review status, and payment notifications.

## Submission and Review

### User Submission

Users must submit:

1. Final report document
2. Video proof
3. Optional supporting photos from weekly uploads

### AI Review

AI reviews report documents only.

AI checks:

1. File exists and is readable.
2. Required template sections are complete.
3. Week 1 research evidence is present.
4. Week 2 build evidence is present.
5. Week 3 testing evidence is present.
6. Week 4 improvement/reflection is present.
7. Content is relevant to the selected project.
8. No obvious unsafe or irrelevant content is detected.

### Manual Review

Video proof is manually reviewed by admin.

Admin can:

1. Approve submission
2. Request resubmission
3. Reject submission
4. Override AI result
5. Issue stars
6. Generate certificate
7. Add reviewer notes

## Stars, Achievements, and Unlocking

1. Completed regular project gives 1 star.
2. Completed Mythical project gives 2–3 stars.
3. Stars are tracked per category.
4. Achievements are issued after accepted project completion.
5. Level unlock depends on age, stars, level, and completed project history.
6. Certificates can be generated after accepted completion.
7. Locked projects must show the lock reason.

Lock reasons:

1. Age locked
2. Level locked
3. Star requirement not met
4. Premium required
5. Coming soon

## Subscription and Payment

### Regular Subscription

1. Family Plan costs EUR 29/month/account.
2. Subscription gives access to regular projects subject to age, level, and unlock rules.
3. Payment may be handled through Flipflop and/or manual admin approval.

### Mythical Premium Project

1. Mythical project is premium expert-led content.
2. MVP pricing reference is EUR 50 per star.
3. A 3-star Mythical project may cost EUR 150.
4. A 2-star Mythical project may cost EUR 100.
5. Premium access activates after payment approval.

### Late Fee

1. Late fee is EUR 0.50 for missed final project deadline.
2. Late fee is capped at EUR 2/month/account.
3. Late fee applies only to the late account.
4. Deadline fee disclosure must be accepted during registration/subscription flow.

## Privacy and Safety

1. Child-related media is private by default.
2. Public display of child project media requires explicit guardian opt-in.
3. Testimonials require explicit parent consent.
4. Experts cannot access child data in MVP.
5. Student ID invitation reduces random public matching.
6. Safety labels are required for every project where relevant.
7. Adult supervision must be shown for cooking, tools, outdoor activities, or risky projects.
8. Account deletion, retention policy, and German privacy/legal pages must be prepared before public launch.

## Acceptance Criteria

The MVP is acceptable when:

1. Public visitor can view Home and About Us.
2. Parent can register with guardian info, child info, category preferences, and consent.
3. Registration requires at least 2 selected categories.
4. User can log in and view My Account.
5. My Account shows child name, Student ID, age, stars, categories, invitations, achievements, and active project.
6. User can update Start Your Career preferences.
7. User can browse project listing by Popular, New Stars, Recommended, and Mythical.
8. Project cards show category, level, age, star reward, description, and availability.
9. Category page shows level navigation and lock/unlock status.
10. Project workspace shows video, tasks, template, team members, invitation input, timer, and progress.
11. Project cannot start with fewer than 2 registered users.
12. User can upload report document and video proof.
13. AI report review can produce passed, failed, or needs manual review status.
14. Admin can manually review video proof.
15. Completion issues stars and achievement.
16. Subscription page shows plan, billing status, payment method, and late fee status.
17. Admin can upload project content.
18. Admin can approve payment.
19. Admin can review submission.
20. All child media is private unless guardian opted into public sharing.
